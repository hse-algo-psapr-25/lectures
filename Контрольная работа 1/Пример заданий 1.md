# Контрольная работа 1. Вариант 1.

## Задание 1
На небольшом деревообрабатывающем предприятии работает цех по изготовлению декоративных рам. Каждая рама сначала проходит фрезеровку на станке №1 (формирование профиля), а затем шлифовку на станке №2. Станки работают независимо, но начать шлифовку рамы можно только после завершения её фрезеровки. Последовательность запуска рам на фрезеровку выбирается произвольно. Время обработки каждой рамы на обоих станках задано и различается. В цех поступил заказ на изготовление 10 декоративных рам по индивидуальным размерам, для каждой из них рассчитано время обработки на первом и втором станках: (6, 7), (4, 5), (3, 1), (2, 1), (6, 5), (2, 3), (5, 4), (6, 8), (4, 3), (3, 2). Необходимо составить расписание работы станков для изготовления указанных декоративных рам за минимально возможное время.

## Задание 3
Команда разработчиков настраивает сборку нового программного продукта, состоящего из 10 этапов, каждый из которых занимает ровно 1 час. Некоторые этапы могут начинаться только после завершения других. Зависимости между этапами заданы следующим списком (запись «X ← Y» означает: этап X можно запустить только после завершения Y):
- Компиляция ядра ← Парсинг конфигурации  
- Сборка UI-модуля ← Парсинг конфигурации  
- Сборка API-модуля ← Парсинг конфигурации  
- Линковка ядра и UI ← Компиляция ядра, Сборка UI-модуля  
- Линковка ядра и API ← Компиляция ядра, Сборка API-модуля  
- Формирование пакета ← Линковка ядра и UI, Линковка ядра и API  
- Выполнение unit-тестов ← Формирование пакета  
- Выполнение интеграционных тестов ← Формирование пакета  
- Статический анализ кода ← Выполнение unit-тестов, Выполнение интеграционных тестов
- Динамический анализ кода ← Выполнение unit-тестов, Выполнение интеграционных тестов

Для выполнения этапов сборки доступны два одинаковых CI-агента (сервера непрерывной интеграции). Каждый агент может выполнять любой этап, но одновременная работа над одним и тем же этапом невозможна, и прерывание начатого этапа недопустимо.

Необходимо составить расписание для выполнения сборки программного продукта за минимально возможное время.
